## Fully complete ✅

- National synthesis (CSV + Parquet + `05_national_hit_rates.png`)

### Data pipeline quality (session 1)
- All 15 sites migrated to `layout: nested`
- `00_verify_setup.ipynb` moved to `notebooks/`
- Colorblind accessibility: `RdYlGn` → `magma`/`viridis` on figs 01–02
- `max_bundle_mb` field on `SiteConfig` (default 150 MB) — fixes `mountain_pass` large-bundle downloads
- Feathered distance-weighted mosaic blending — re-run for 5 worst-seam sites (Stillwater, Tonopah, Jerritt Canyon, Bear Lodge, Oatman)
- Global colorbar standardization: `compute_global_limits()` in `pipeline.py`; `--no-global-limits` flag preserves per-site behavior. Limits: Silica 0.884–0.979 · Carbonate 1.130–1.445 · Mafic 0.692–0.885
- North arrow + scale bar on fig 03; UTM ticks → km; dense-deposit readability; title overlap fixed
- Figure 05 national stacked bar (Earth MRI categories, Paul Tol palette) — migrated from notebook-only to `synthesis.py`

### Structure data — all 15 sites (session 2)
- 13 sites: USGS Quaternary Faults (QFAULTS REST API)
- Bear Lodge + Jerome: USGS SGMC FeatureServer (all-age faults); `scripts/download_sgmc_structures.py`
- `StructureLayer`: `label` field + `display_label()` method
- `n_deposits_on_structure` NaN fix: was emitting `0.0` for no-structure sites, now `None`
- Stillwater bbox bug fixed: fault GeoJSON was covering Nevada instead of Montana; replaced with correct SGMC data
- `docs/architecture.md`, `docs/roadmap.md`, `docs/structure_layers.md` updated

### Figure quality (session 2)
- **Hillshade gray-box bug** (Silver Peak, Yerington, Tonopah, Steamboat Springs, Goldfield): axes limits now restored after contextily (not before), eliminating tile-grid expansion artifact
- **Hillshade NaN transparency**: `terrain.py` returns float32 with NaN nodata; consumers use `set_bad(alpha=0.0)` — DEM gaps show basemap through
- **Figure 03 — amber fault corridor buffer fill**: `unary_union` polygon (`#e67e22`, alpha=0.12) replaces raw gray lines; legend entry: `"Structural corridor (±500 m)"`
- **Figure 03 — on-structure fraction annotation**: `ax.text` bottom-left: `"32/58 deposits (55%) within 500 m of structure"`
- **Bbox coordinate stamps**: `"41.80°–42.40°N · 118.10°–117.30°W"` on all figures (figs 00–02 as `fig.text`; fig 03 as title third line)

### Scalability layer 1 (session 2)
- **`--skip-existing`** flag: skips site if `03_deposit_overlay.png` + provenance JSON exist
- **`--workers N`** parallel batch runner: `ProcessPoolExecutor`; 15 sites in 90 s with 4 workers (was ~6 min sequential)
- **`figures/index.html`**: sortable grid of 15 site cards (fig 03 thumbnails + metrics); no external deps; regenerated by `synthesize`
- **`results/results.duckdb`**: `site_summaries` table written by `synthesize`; SQL-queryable
- **Figure 06** (`figures/06_structure_hit_rate.png`): log-scale scatter of mean fault distance vs hit rate, sized by deposit count
- Structure GeoJSON loaded once per run (was two disk reads + CRS reprojects); `structure_buffer_union()` helper; missing-file guard

### Structure data + metrics (session 3)
- **Figure `05_structure_proximity.png`**: strip chart per commodity group, log x = `nearest_structure_m`, gold stars = in-zone, steelblue dots = out-of-zone, dashed threshold line. Written by `save_structure_proximity_figure()` in `pipeline.py`. Generated for all sites with structure data.
- **Sub-row structure metrics**: `n_deposits_on_structure` + `mean_nearest_structure_m` now populated on `commodity` / `earth_mri` / `mineral_system` CSV rows (was always NaN). Added `annotated_deposits` param to `compute_site_summary()` in `metrics.py`.
- **Lazy SGMC auto-fetch** (`auto_fetch_structure()` in `pipeline.py`): when `structure_layers` is empty, auto-queries USGS Quaternary Faults API then falls back to SGMC FeatureServer; caches to `data/structures/{site_id}_faults_auto.geojson`. New sites get structure annotation automatically on first run without any manual download step.

## Deferred — requires scientific judgment or significant effort 🔬

| Item | Notes |
|---|---|
| Scan-line noise (Bear Lodge, Goldfield, Darwin, Marysvale) | ASTER whiskbroom artifact. Requires destriping (moment matching per detector). Cosmetic; does not affect classification. |
| McDermitt blanket anomaly coverage | Zones cover >30% of bbox — pervasive Li-clay caldera lithology OR `strong_score_min` too loose. Needs geoscience call. |
| Residual seam artifacts (Stillwater, Tonopah) | Feathered blending improved but didn't eliminate. Options: seasonal filtering, cloud masking, more aggressive overlap weighting. |
| Coverage gaps (Silver Peak 78%, Jerome 89%) | ASTER swath rotation at mid-latitudes — fundamental sensor constraint, not fixable with mosaicking. Bboxes tightened. |

## Quick-reference commands

```bash
# Run one site
conda run -n aster-minerals python -m critical_minerals_aster run --site mcdermitt

# Run all 15 in parallel (4 workers)
conda run -n aster-minerals python -m critical_minerals_aster run-batch --all --workers 4

# Skip already-processed sites
conda run -n aster-minerals python -m critical_minerals_aster run-batch --all --workers 4 --skip-existing

# Regenerate national summary + all synthesis figures
conda run -n aster-minerals python scripts/synthesize_national.py

# Download SGMC structure data (bear_lodge, jerome, or new sites)
conda run -n aster-minerals python scripts/download_sgmc_structures.py

# Download Quaternary fault data
conda run -n aster-minerals python scripts/download_usgs_faults.py

# Query results
conda run -n aster-minerals python -c "
import duckdb
con = duckdb.connect('results/results.duckdb')
print(con.execute(\"SELECT site_id, hit_rate_pct, n_deposits_on_structure FROM site_summaries WHERE row_type='site' ORDER BY hit_rate_pct DESC\").fetchdf())
"
```
